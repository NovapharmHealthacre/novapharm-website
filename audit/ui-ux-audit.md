# UI and UX Audit

## Source-Level Findings Resolved

- Rebuilt the public experience around a pharmaceutical operations image, restrained typography, clear maturity labels and evidence-led navigation.
- Added the complete corporate, governance, leadership, service, regulatory, portfolio, partnership, technology, Insights, account and legal journeys.
- Added responsive navigation with escape, outside-click and focus behaviour.
- Added accessible Contact and four-stage account-application forms with status regions, error summaries and submission locking.
- Added explicit Customer, Employee and Board login choices; the administrator model supports all four scopes.
- Kept all public portal files data-free and placed operational content behind server authentication.
- Added stable grids, text wrapping, table overflow containment, reduced-motion support, visible focus and dark-mode compatibility.
- Applied the byte-identical approved SVG/PNG identity throughout public, portal, email and structured-data surfaces.

## Repository Checks Passed

All 33 intended public pages and 40 protected shells have mobile viewport declarations. Every image has alternative text and intrinsic dimensions; heading order, one-H1 structure, skip links, canonicals, JSON-LD and local assets pass validation. Generated public pages contain no placeholder copy or retired raw browser errors.

## Rendered Acceptance Status

Rendered acceptance is not complete. The sandbox rejects a local server, the in-app browser blocks local-file navigation by policy, no WebKit/Safari-compatible engine is available, and no owner-approved private preview exists. The exact seven-viewport matrix, pages, inspection criteria and honest `Not run` status are recorded in `audit/visual-acceptance-report.md`. No visual-perfect, Lighthouse or WCAG-conformance claim is made until that matrix passes.
